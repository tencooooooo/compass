# Development Principles

Compass is designed for long-term maintenance. Development should favor clarity, small steps, and durable documentation.

## Build Small

Add one clear capability at a time. Each version should be understandable, testable, and reversible.

## Preserve Backward Compatibility

Avoid breaking existing file paths, report formats, and workflows unless a migration is planned and documented.

## Keep Knowledge Current

When analysis rules, project assumptions, definitions, or design decisions change, update `knowledge/`.

## Keep README Current

When user-facing behavior changes, update `README.md` so a new contributor can understand the project from the repository front page.

## Separate Prompts From Code

AI prompts should live in `prompts/`, not inside Python files. This makes analysis behavior easier to review and improve.

## Prefer Facts First

Reports should begin with available facts before adding interpretation. Missing data should be shown honestly.

## Avoid Hidden Complexity

Use simple structures until real complexity appears. Over-designed patterns should be avoided unless they make future changes easier.

## Protect Generated Data

Generated outputs such as raw storage, events, logs, and reports are excluded from Git and should be regenerated by GitHub Actions or local scripts.
